# 0001. GitHub workflow для задач через ChatGPT/Codex

Дата: 2026-06-07

## Решение

Работу вести через GitHub Issues и Pull Requests.

Каждая задача должна иметь:

- цель;
- ссылки на документы или файлы;
- ожидаемый результат;
- критерии готовности;
- ограничения по нормативной ответственности.

## Причина

Такой формат позволяет давать ChatGPT/Codex конкретные задания, отслеживать историю решений и проверять изменения перед включением в основную ветку.

## Следствия

- Для новых работ создаются Issues.
- Для изменений кода создаются Pull Requests.
- Документы `docs/TZ.md`, `docs/sp20-loads.md` и `docs/ml-dataset.md` считаются источниками требований.
