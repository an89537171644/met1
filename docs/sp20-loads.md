# Список необходимых нагрузок и исходных данных по СП 20 для плоской двухскатной однопролётной рамы

**Дата:** 2026-06-07  
**Объект:** плоская двухскатная однопролётная стальная рама.  
**Грузовая полоса:** шаг рам `B`, м.  
**Координаты:** X — по пролёту, Y — вертикально. В MVP учитывается ветер в плоскости поперечной рамы `+X/-X`.

---

## 1. Обязательные общие данные

| № | Данные | Обязательно | Назначение |
|---|---|---:|---|
| 1 | Город/район строительства | да | Определение климатических параметров |
| 2 | Редакция СП 20 и номер изменений | да | Воспроизводимость расчёта |
| 3 | Назначение здания | да | Выбор полезных/технологических нагрузок |
| 4 | Уровень ответственности / класс сооружения | желательно | Для общей расчётной постановки |
| 5 | Срок службы / расчётная ситуация | желательно | Для комбинаций и особых ситуаций |
| 6 | Шаг рам | да | Перевод поверхностных нагрузок в линейные на раму |
| 7 | Длина здания | да | Для ветра, температурных эффектов и отчёта |
| 8 | Геометрия кровли | да | Снеговые и ветровые коэффициенты |
| 9 | Тип местности для ветра | да | Расчёт ветрового профиля по высоте |
| 10 | Наличие проёмов/внутреннего давления | условно | Внутреннее давление/отсос |

---

## 2. Постоянные нагрузки `G`

| Load ID | Что учитывать | Единица исходных данных | Как прикладывать к раме | Обязательность |
|---|---|---:|---|---|
| G1 | Собственный вес стальных стоек и ригелей | кг/м или кН/м | Автоматически по выбранному сечению | да |
| G2 | Кровельный пирог: профлист/настил, утеплитель, гидроизоляция, пароизоляция | кН/м² | На наклонные ригели через грузовую полосу | да |
| G3 | Прогоны кровли | кН/м² или кН/м | На ригели | да |
| G4 | Связи покрытия и вертикальные связи | кН/м² или отдельная масса | В постоянную нагрузку | желательно |
| G5 | Фахверк/стеновые прогоны/ограждения | кН/м² | На стойки/узлы | если есть |
| G6 | Подвесные инженерные системы | кН/м² или кН | На ригели/узлы | если есть |
| G7 | Огнезащита/антикоррозионные покрытия | кг/м² или кг/м | В постоянную нагрузку | если есть |
| G8 | Стационарное оборудование | кН | На узлы/элементы | если есть |

---

## 3. Снеговые нагрузки `S`

| Load ID | Схема | Что нужно задать | Комментарий |
|---|---|---|---|
| S0 | Базовые климатические данные | район/город или нормативное значение веса снегового покрова | Хранить источник значения |
| S1 | Равномерная снеговая нагрузка на оба ската | уклон кровли, коэффициенты формы по СП 20, грузовая полоса | Обязательная схема |
| S2 | Несимметричная снеговая нагрузка: левый скат больше/правый меньше | параметры схемы для двухскатной кровли | Обязательная для проверки опасных сочетаний |
| S3 | Несимметричная снеговая нагрузка: правый скат больше/левый меньше | параметры схемы для двухскатной кровли | Зеркальная к S2 |
| S4 | Снеговые мешки/заносы у перепадов, парапетов, фонарей | геометрия препятствий | Если есть перепады, парапеты, фонари |
| S5 | Пониженная снеговая нагрузка | нормативные условия применения | Только если обосновано по СП 20 |

Для обучения нейросети важно сохранять не только итоговую нагрузку, но и исходные параметры: район, нормативное значение, уклон, схему, коэффициенты и применённую редакцию СП.

---

## 4. Ветровые нагрузки `W`

| Load ID | Схема | Что нужно задать | Комментарий |
|---|---|---|---|
| W0 | Базовые климатические данные | район/город или нормативное значение ветрового давления | Хранить источник значения |
| W1 | Ветер `+X` слева направо | тип местности, высота здания, аэродинамические коэффициенты стен и кровли | Обязательная схема для плоской рамы |
| W2 | Ветер `-X` справа налево | те же параметры | Обязательная зеркальная схема |
| W3 | Внутреннее давление `+ci` | проёмы/герметичность/условия внутреннего давления | Если требуется |
| W4 | Внутреннее давление `-ci` | то же | Если требуется |
| W5 | Пульсационная составляющая | высота, динамические характеристики, нормативная методика | Если требуется по условиям СП 20 |
| W6 | Ветер вдоль здания | пространственная работа/связи | Не входит в 2D MVP, но оставить поле |

Для 2D MVP ветровую нагрузку нужно приводить к линейным нагрузкам на стойки и ригели с учётом грузовой ширины `B`.

## 5. Полезные, ремонтные и технологические нагрузки `L/Q`

| Load ID | Нагрузка | Обязательность | Комментарий |
|---|---|---|---|
| Q1 | Ремонтная/эксплуатационная нагрузка на кровлю | желательно | Для обслуживания кровли, если предусмотрено |
| Q2 | Нагрузки от людей на площадках/мостиках | если есть | Для обслуживаемых зон |
| Q3 | Сосредоточенные нагрузки | если есть | Например подвесы, оборудование |
| Q4 | Складируемые материалы | если есть | Если здание имеет складскую функцию |
| Q5 | Транспортные средства | если есть | Обычно не для самой поперечной рамы покрытия |

---

## 6. Крановые нагрузки `CR`

Крановые нагрузки включать только если в задании есть мостовой или подвесной кран.

| Load ID | Что задать | Комментарий |
|---|---|---|
| CR1 | Тип крана: мостовой/подвесной | Влияет на схему нагрузки |
| CR2 | Грузоподъёмность | Из паспорта крана |
| CR3 | Нагрузки на колёса/катки | Для расчёта подкрановых конструкций и рамы |
| CR4 | Вертикальные воздействия | На колонны/ригели через крановые пути |
| CR5 | Горизонтальные поперечные воздействия | Для рамы в плоскости |
| CR6 | Горизонтальные продольные воздействия | Для связевой системы, не для 2D MVP |
| CR7 | Динамические коэффициенты | По СП 20 и паспорту крана |

Если кранов нет, модуль должен явно записывать `cranes: false`.

## 7. Температурные, гололёдные и особые воздействия

| Группа | Когда учитывать | Что хранить |
|---|---|---|
| T — температура | Если здание длинное, есть жёсткие связи/температурные ограничения или требуется заданием | min/max температура, монтажная температура, схема закреплений |
| I — гололёд | Для открытых или подверженных обледенению конструкций | район гололёда, толщина стенки, коэффициенты |
| A — особые воздействия | Если задано: аварии, удар, пожар, сейсмика, отказ элемента | отдельные расчётные ситуации и сочетания |
| C — монтажные нагрузки | Если нужна стадийность монтажа | временные схемы опирания, временные нагрузки |

## 8. Минимальный набор нагрузочных случаев для MVP

```yaml
load_cases:
  - G_self_weight
  - G_roof_dead
  - G_purlins_bracing
  - S_symmetric
  - S_left_unbalanced
  - S_right_unbalanced
  - W_plus_x_external
  - W_minus_x_external
  - W_plus_x_internal_positive
  - W_plus_x_internal_negative
  - W_minus_x_internal_positive
  - W_minus_x_internal_negative
  - Q_roof_maintenance
```

Дополнительно, если есть:

```yaml
optional_load_cases:
  - T_positive
  - T_negative
  - I_ice
  - CR_crane_vertical
  - CR_crane_horizontal_transverse
  - A_special
  - C_construction_stage
```

## 9. Сочетания нагрузок

Модуль сочетаний должен формировать минимум:

1. Основные сочетания для первой группы предельных состояний.
2. Сочетания для второй группы предельных состояний: прогибы и перемещения.
3. Специальные сочетания, если есть особые воздействия.
4. Отдельные сочетания с ведущей снеговой нагрузкой.
5. Отдельные сочетания с ведущей ветровой нагрузкой.
6. Отдельные сочетания с несимметричным снегом.
7. Зеркальные сочетания для ветра `+X` и `-X`.

В таблицах сочетаний обязательно хранить:

| Поле | Описание |
|---|---|
| `combo_id` | Уникальный идентификатор сочетания |
| `limit_state` | ULS/SLS/special |
| `leading_load` | Ведущая переменная нагрузка |
| `load_id` | Нагрузочный случай |
| `factor` | Коэффициент в сочетании |
| `source_norm` | Норма и редакция |
| `notes` | Комментарий |

## 10. Данные, которые обязательно сохранять для нейросети

Для каждого кейса:

1. Геометрия рамы.
2. Материал.
3. Сортамент и выбранный профиль.
4. Все нагрузки по группам.
5. Все сочетания.
6. Версия ЛИРА.
7. Версия СП 16 и СП 20.
8. Усилия по элементам и сочетаниям.
9. Перемещения.
10. Коэффициенты использования.
11. Признак `pass/fail`.
12. Эталонное минимальное сечение.
13. История перебора сечений, если доступна.
