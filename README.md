# Steel Frame Designer

Программа для подбора единого стального сечения стоек и наклонных ригелей плоской двухскатной однопролетной рамы.

Проект ведется как магистерская работа. Репозиторий подготовлен так, чтобы задачи можно было ставить через GitHub Issues и выполнять с помощью ChatGPT/Codex.

## Цель

Собрать MVP, который:

- принимает геометрию плоской двухскатной рамы;
- хранит нагрузки и сочетания по СП 20.13330.2016;
- импортирует результаты расчетов из ЛИРА-САПР / ПК ЛИРА;
- формирует обучающий датасет;
- обучает baseline-модель для top-N рекомендации сечений;
- проверяет рекомендованное сечение перед отчетом;
- формирует отчет Markdown/CSV.

## Нормативная база

- СП 16.13330.2017 "Стальные конструкции";
- СП 20.13330.2016 "Нагрузки и воздействия".

В данных обязательно хранить редакцию и номер изменения норм.

## Данные

Шаблоны CSV лежат в `data/templates/`.

Сейчас это демонстрационные шаблоны, а не готовая расчетная база. Для обучения модели нужно заполнить реальные результаты ЛИРА, сортамент и метки выбранных сечений.

## Как ставить задачи для ChatGPT/Codex

1. Создать GitHub Issue по шаблону `Engineering task`.
2. В описании указать цель, входные файлы, ожидаемый результат и критерии готовности.
3. Для работы в чате дать ссылку на Issue или скопировать его текст.
4. После выполнения проверять изменения через Pull Request.

Пример задачи:

```text
Реализовать Pydantic-модель FrameGeometry и тесты валидации.
Опирайся на docs/TZ.md и config.example.yaml.
```

## CLI

```powershell
steel-frame-designer init
steel-frame-designer doctor
steel-frame-designer validate-input
steel-frame-designer build-loads
steel-frame-designer import-lira data/templates/lira_element_forces.csv
steel-frame-designer report
```

`config.example.yaml` является шаблоном. Команда `validate-input` показывает `ERROR` для обязательных пустых полей нагрузок и возвращает ненулевой код, пока эти поля не заполнены.

## Локальный старт

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -e ".[dev]"
pytest
```

Пока кодовая часть является каркасом. Полный инженерный расчет по СП 16 в MVP не заявляется без отдельной верификации.
