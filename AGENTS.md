# AGENTS

Инструкции для ChatGPT/Codex при работе с этим репозиторием.

## Контекст проекта

Проект: подбор единого сечения для плоской двухскатной однопролетной стальной рамы.

Основные документы перед началом работы:

1. `docs/TZ.md`
2. `docs/sp20-loads.md`
3. `docs/ml-dataset.md`
4. `docs/backlog.md`
5. `config.example.yaml`

## Правила работы

- Не считать нейросеть нормативным расчетом.
- Любая рекомендация ML должна проходить проверку расчетным модулем или ЛИРА.
- Не зашивать нормативные коэффициенты магическими числами в код.
- Для коэффициентов хранить источник: норма, редакция, пункт/таблица, дата.
- Не смешивать данные разных редакций СП без отдельного признака версии.
- Исходные экспорты ЛИРА хранить без изменения.
- Все импорты должны проверять единицы измерения и обязательные поля.
- Для MVP допустимы заглушки инженерных проверок, но они должны явно называться заглушками.

## Стиль разработки

- Python 3.11+.
- Предпочитать Pydantic-модели для входных данных.
- CSV использовать как базовый формат обмена.
- Для больших расчетных таблиц позже можно добавить Parquet.
- Тесты добавлять рядом с новой логикой.
- Документацию обновлять вместе с изменением структуры данных или поведения CLI.

## Definition of Done для задач

Задача считается готовой, если:

- изменены только относящиеся к задаче файлы;
- есть тесты или явно описано, почему они не нужны;
- `pytest` проходит;
- обновлена документация, если изменился пользовательский сценарий;
- в финальном ответе перечислены изменения, проверки и ограничения.
